# 1.概述
本接口设计文档涵盖了基于面部表情疲劳项目中的核心接口，主要用于处理人脸疲劳检测相关的视频数据上传以及实时推送处理结果。这些接口通过 HTTP 和 WebSocket 协议实现，确保了前端、后端以及模型之间的数据交互和实时通信。
# 2.接口列表
## 2.1 前后端接口
### 2.1.1 视频上传接口
   接口名称：video_upload
   接口地址：POST /faceDetectService/video_upload
   功能描述：接收前端上传的视频文件和用户 ID，将视频流传递给后端进行处理。
   请求参数：

| 参数名    | 类型            | 是否必传 | 描述      |
|--------|---------------|------|---------|
| file   | MultipartFile | 是    | 上传的视频文件 |
| userId | String        | 是    | 用户 ID   |
返回参数：

| 参数名  | 类型      | 描述                  |
|------|---------|---------------------|
| code | Integer | 状态码，200 表示成功，其他表示失败 |
| msg  | String  | 消息描述                |
### 2.1.2 疲劳检测结果实时推送接口
接口名称：face_fatigue_push
接口地址：WebSocket /topic/face_fatigue/{userId}
功能描述：将模型服务器处理后的疲劳等级和疲劳指数实时推送给客户端。
后端消息发送参数说明：
| 参数名          | 类型     | 描述                   |
|--------------|--------|----------------------|
| userId       | String | 用户 ID                |
| fatigueRank  | String | 疲劳等级，取值范围为 "0" - "3" |
| fatigueIndex | String | 疲劳指数，取值范围为 0 - 10    |
| timestamp    | Long   | 消息发送的时间戳             |
| emotionCat   |
